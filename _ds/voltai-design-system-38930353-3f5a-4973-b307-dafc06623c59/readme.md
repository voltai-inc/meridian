# Voltai Design System

Brand & UI system for **Voltai** — a Stanford-backed AI-infrastructure company building **Project Concord**, a $35B, 3-gigawatt data center in Jeollanam-do, South Korea, designed, built, and operated entirely by AI. Partnered with investor group **Stock Farm Road**. The system serves the B2B/investor pitch deck and the wider brand surface.

**Aesthetic:** monochrome technical, in the lineage of Palantir, Anduril, and Nominal decks. Near-black ink on warm off-white paper, one restrained green accent used **once per surface**, hairline structure, hard corners only. Precise, restrained, premium — engineering document, not marketing brochure.

## Sources

No codebase or Figma was provided. The system was derived from a written brief plus three reference screenshots (in `uploads/`):
- **Palantir "HyperAuto on AWS"** title slide — the warp-grid hero treatment (rated 9/10 by the client; the signature motif).
- **Nominal** mobile site — schematic node diagrams, mono labels, thin frames, dashed green control loop.
- **Nominal** hero — Space-Grotesk-style display headline + mono "REQUEST DEMO" CTA.

## CONTENT FUNDAMENTALS

**Voice:** an engineering spec, not a sales page. Declarative, exact, confident. State capability as fact ("Operated by AI. 3 gigawatts."). No hype words, no exclamation, no hedging.

**Person:** third-person and system-oriented ("Project Concord is…", "the control loop senses…"). Avoid "we" / "you" in structural copy; first-person plural is acceptable only in a direct investor address.

**Casing:**
- **Headlines / display** — sentence case, short, often a hard statement with a period: *"No operators on the critical path."*
- **Labels, captions, data, tags, callouts** — ALL CAPS, IBM Plex Mono, wide tracking: `FIG. 02 — OPERATING TOPOLOGY`, `VLT-3GW-0427`, `STATUS NOMINAL`, `26·03·26`.

**Engineering callouts:** structural labels read like schematic annotations, never decoration. Figure numbers (`FIG. 01`), mono node IDs (`VLT-3GW-0427`), system tags (`CONCORD / OPS`), and dotted dates (`26·03·26`). These carry meaning and wayfinding, not flavor.

**Numbers:** precise and unit-tagged — `3.0 GW`, `35 BN USD`, `0 OPERATORS`, `24/7`. Big numbers in display weight, units in mono.

**Emoji:** never. **Exclamation marks:** never. **Unicode marks** used sparingly and technically: `·` (mid-dot separators), `—` / `———` (rules), `›` (forward action), `●` (status). 

**Examples**
- ✅ "A 3-gigawatt data center designed, built, and operated entirely by AI."
- ✅ "Power, cooling, and compute are sensed and actuated by a continuous AI control loop."
- ❌ "Revolutionizing the future of data centers! 🚀"
- ❌ "We're so excited to share our amazing journey."

## VISUAL FOUNDATIONS

**Color.** Monochrome warm-neutral ink ramp (`--ink-900 #0A0A0A` → hairlines) on warm paper (`--paper-0 #F4F4F2`). An inverted **void** ramp (`#0A0A0A`→`#1E1E1C`) for dark schematic surfaces. Exactly **one** accent: **signal green `#00B894`** (`#008E72` as AA text on paper), used a single time per surface — the control loop, one node, or one state. Never two greens on one surface. No other hues, ever.

**Type.** Display + body: **Space Grotesk** (700 for headlines with tight `-0.03em` tracking; 400 for running body). Labels/data/callouts: **IBM Plex Mono** (500, UPPERCASE, wide `+0.14em` tracking). The mono/display contrast IS the brand — a machined caption against a confident headline.

**Backgrounds.** Flat paper or flat void — **no gradients, no photography, no texture/grain**. The only "imagery" is generated line-work: the warp grid and schematic diagrams. Full-bleed warp fields are used as hero bands and section dividers.

**Signature motif — the warp grid.** A fine black line lattice that pinches toward attractor points, producing a gravitational vortex. Generated by `assets/warp-grid.js` (`VoltaiWarp.mount(canvas, {cells, attractors})`). Sparse node dots are scattered across it; exactly one is green.

**Schematic diagrams.** Wired node boxes (hard-cornered, hairline border) connected by **right-angle** connectors. A single **dashed green "control loop"** (`stroke-dasharray: 6 6`) expresses feedback/automation. Solid gray = data path; dashed green = control.

**Structure.** Every primary surface (slide, hero) carries the **inset hairline frame** — a 1px rule inset ~20–28px from the edge. All structure is expressed through hairline rules and frames, **never shadows**.

**Corners & radius.** `--radius: 0`. Hard corners everywhere — buttons, cards, tags, nodes, frames. Non-negotiable.

**Shadows.** None. No drop shadows, no inner shadows, no glows. Depth is communicated by rules and surface tint only.

**Cards.** A card is a flat surface (`--paper-2 #FBFBFA`) with a 1px border (`--ink-200` soft, or `--ink-900` strong) and zero radius. No elevation.

**Borders / rules.** `--border-thin` (1px `--ink-900`) for strong structure; `--border-soft` (1px `--ink-200`) for dividers; `--border-void` on dark. Hairlines do all the work.

**Spacing.** 4px base, mechanical even steps (`--space-1` … `--space-10` = 4→128px).

**Motion.** Restrained and machined — `--ease-mech: cubic-bezier(0.2,0,0.1,1)`, 120–200ms. Crisp slides and fades only. **No bounce, no spring, no decorative loops.** Slide content animates in from hidden; the visible end-state is the base style (so print/PDF/reduced-motion show content).

**Hover states.** Bordered (secondary) buttons **fill ink** on hover (text → paper); solid buttons darken to `--ink-700`; ghost text shifts to `--signal-text`. Tags are static. No scale on hover.

**Press states.** No shrink/scale. State change is communicated by the ink fill, not motion.

**Transparency / blur.** Effectively none. Surfaces are opaque; no frosted glass, no protection gradients. Contrast comes from paper-vs-void, not scrims.

**Layout rules.** Frame + header (logo left / mono tag right) + content + hairline-ruled mono footer (`SYSTEM TAG ——— PAGE / TOTAL`). Slides are 1280×720 (16:9), letterboxed on black.

## ICONOGRAPHY

Iconography is **minimal and geometric**, drawn from the same primitives as the schematic system — squares, hairline strokes, right angles, the occasional filled accent square. There is **no icon font and no large icon set**; iconography is intentionally sparse.

- **The logo mark** (`assets/logo-mark.svg`) is a "warp node" — four wires pinching to a single green attractor square. It doubles as the brand's only persistent glyph.
- **Schematic node icons** are tiny hairline squares (18px) inside node boxes; the active/control node gets a filled green square.
- **Status / structural marks** use Unicode technically: `●` status, `·` separators, `›` forward, `—`/`———` rules. No decorative pictograms.
- **Emoji: never.** No multicolor icons, no rounded "friendly" iconography.

If a project genuinely needs UI utility icons (e.g. a product surface), substitute **Lucide** (CDN) at 1.5px stroke, square caps, recolored to `--ink-900` — it matches the hairline geometry. **This would be a substitution — flag it and confirm before relying on it.**

**Assets in `assets/`:** `logo-mark.svg`, `logo-lockup.svg`, `logo-lockup-inverse.svg`, `warp-grid.js`.

## SUBSTITUTIONS / CAVEATS

- **Fonts** load from Google Fonts (`tokens/fonts.css`) — Space Grotesk + IBM Plex Mono are exact matches to the brief, not substitutes. If self-hosted/offline binaries are required, swap the `@import` for local `@font-face` rules. *(Flagged to client.)*
- No codebase/Figma was available, so there is no product UI kit yet — only the brand foundations, components, and the pitch-deck template. Provide a codebase or Figma to build product surfaces.
